Newsletter

"Extendable Newsletter Module for the Zikula Application Framework"

 

Features:

  • XHTML 1.0 Strict Compliant
  • Database or flat file archiving
  • Unlimited Extendability through the use of Smarty Plugins
  • Option: Anonymous subscriptions
  • Option: auto-approve subscriptions
  • Newsletter types: text, html, text with link
  • Option: allow frequency changes (weekly, monthly or yearly subscriptions)
  • Option: send date - day of the week to send (automatically)
  • View Archives
  • Automatic pruning of archives
  • Able to send to single and multiple newsletters to chosen recipients
  • User import
  • update check

 

Requirements:

  • Zikula CMS (v.1.0.0+)
  • pnRender Module (core)
  • Mailer Module (core)
  • Xanthia theme

 

Before Installing:

  • Read the license.
  • Move plugins from /Newsletter/pndocs/your_theme/plugins/ into their relative directories.
  • Add this line above the body tag of your Xanthia theme

    <!--[newsletter_maintenance]-->

 

Install:

  • Upload into modules directory
  • Initialize and Activate from modules admin
  • Configure default settings in the Newsletter admin area. (see configuration.txt for more details)
  • Modify Newsletter/pntemplates/newsletter_template_*.htm templates accordingly
  • Activate Newsletter/Maintenance block. No output is displayed.
    Its purpose is to send queued mailers on the "Send Day" and expire "stale" archives.
  • Link module in main menu as {Newsletter} or via URL as index.php?module=Newsletter
  • Mark West's port of the Xaraya scheduler module IS supported. (see the configuration.txt file)

 

Send via cron task and wget

  • Set your private "Admin Key" (automatically generated on install/upgrade) in Admin->Newsletter
  • Set up a crontab to run the send process (see below).
dhayes$: crontab -e
*/10 */2 * * 2 /usr/local/bin/wget --quiet --delete-after "http://www.yoursitename.com/index.php?module=Newsletter&func=send&code=ADMIN_KEY"

this particular task runs tuesday (as set in my configuration), every other hour in 10 minutes intervals. quietly and deletes the returned file after processing. More about cron tabs .. to over-ride the send_day as set in the admin add the parameter "scheduled":

*/10 */2 * * 2 /usr/local/bin/wget --quiet --delete-after "http://www.yoursitename.com/index.php?module=Newsletter&func=send&code=ADMIN_KEY&scheduled=1"

this will allow you to over-ride the send day as set in the admin area (not recommended, but whatever) .. this new "feature" also returns the amount of emails sent (per the admin configuration "Send Per Request").
